レガシーCGIゲームのCSS分離＆レスポンシブ化指示書
プログラミング指示

    あなたはフロントエンドおよびUI設計のスペシャリストです。
    現在1ファイルに全てのスタイルが詰め込まれており、テーブルレイアウト（<table>によるページ構築）で非レスポンシブな状態にあるレガシーなCGIゲームのHTMLテンプレートとCSSを、以下の設計規則に従ってモダンなレスポンシブデザインへリファクタリングしてください。
    1. 最終的なCSS分離構成

    スタイルは以下の3つのファイルに厳格に分離し、それぞれの役割のみを記述してください。

        common.css (全システム共通の土台)

            フォント定義、CSSリセット。

            CSSカスタム変数（カラーパレット、余白、角丸の定義）。

            サイト全体に共通するUI（トースト通知、ダイアログ/モーダル、スピナー、フッターなど）。

        game.css (ゲーム個別・各パーツの基本デザイン)

            共通レイアウト（.layout-stack, .layout-cluster, .layout-grid など）。

            共通コンポーネント（.card, .btn, .form-control などの基本スタイル）。

            各個別画面（トップページ、マイページ、戦闘画面、ショップ等）のデスクトップ基準レイアウト。

            ※このファイル内には @media（メディアクエリ）を書かないでください。

        mobile.css (モバイル専用補正スタイル)

            メディアクエリ @media screen and (max-width: 768px) を基準とします。

            PC表示をモバイルで縦積みにする、画像を縮小する、タップ領域を広げるなどの「上書き（補正）スタイル」のみを記述してください。

    2. リファクタリング実行手順（4つのフェーズ）
    【Phase 1】HTMLテンプレートの構造変換（テーブルレイアウトからの脱却）

    既存のHTMLテンプレートから、ページレイアウト目的で使用されている <table> を廃止し、セマンティックなタグ（<header>, <main>, <footer>, <section>）と <div> に置き換えてください。

        配置（レイアウト）目的のテーブル: Flexbox（.layout-stack / .layout-cluster）や Grid（.layout-grid）に置き換えます。

        データ表示（ステータス等）目的のテーブル: <table> のまま残して構いませんが、モバイルで横スクロールできるよう .table-wrapper { overflow-x: auto; } で囲むなどの対策を施してください。

        古い属性の排除: HTMLタグに直接書かれている width="800"、border="0"、align="center"、cellpadding 等の装飾属性はすべて削除し、CSS側で制御するように修正してください。

    【Phase 2】common.css の土台作成と変数定義

    サイト全体で一貫したトーン＆マナーを保つため、色や余白を変数として定義してください。
    code CSS

:root {
  /* メインカラー、サブカラー、ネオン、背景、暗色 */
  --color-primary: #xxxxxx;
  --color-background: #xxxxxx;
  /* 余白サイズ（大・中・小） */
  --space-s: 5px;
  --space-m: 10px;
  --space-l: 20px;
  /* 角丸 */
  --radius-m: 8px;
}

これと合わせて、リセットCSSとグローバルなリンクカラー、共通フォントを common.css に定義してください。
【Phase 3】game.css（基本コンポーネント）の設計

common.css の変数を使用し、再利用可能なパーツ（コンポーネント）をクラスとして定義してください。

    レイアウトクラス群 (Layout):

        .layout-stack: 子要素を縦に積み上げる（flex-direction: column）。

        .layout-cluster: 子要素を横並びにし、端で折り返す（flex-wrap: wrap）。

    幅制限クラス群 (Width System):

        .content-sm (最大幅 600px 程度、左右中央寄せ) などを設定し、各画面の幅を制御。

    コンポーネント群 (Component):

        .card (枠線、背景、角丸を持つボックス。.card__header, .card__body 等の内包要素を含む)。

        .btn (枠線、3Dライクな影、ホバー・アクティブ・ disabled 状態を持つボタン)。

        .form-control (共通の入力フォーム、フォーカス時の青枠ハイライト)。

【Phase 4】mobile.css（メディアクエリ）の実装

モバイル環境（幅 768px 以下）で操作性が低下しないよう、mobile.css 内に上書き用の記述を追加してください。

    PCで横並びになっている要素を、縦積み（flex-direction: column や grid-template-columns: 1fr）に変更。

    ボタン（.btn）などのタップターゲットの最小高さを確保（タップしやすくするため）。

    不要なマージンの削減、フォントサイズの微調整。

3. JavaScript（JS）との結合の保護（最重要）

HTML内の特定の要素（クラス名やID）は、ゲーム内の機能（タイマー制御、データ送信、並び替え機能など）を実行するJavaScriptのセレクタとして使用されています。

    JSが要素特定のために参照しているクラス（例: .js-trigger など）やID名（例: #bbs）は絶対に削除・改変しないでください。

    見た目の変更やレスポンシブ対応は、それらのJS依存クラスに直接スタイルを肉付けするのではなく、上記で定義した共通コンポーネントクラス（.card やレイアウトクラス等）を別途HTMLに併記することで実現してください。
