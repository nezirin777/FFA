本スクリプトは、Perl版のゲームCGI「FFA いく改」を Python 3 / CGI 環境へ移植・改修したものです。
既存の Perl スクリプトとは独立して、`FFA_py` フォルダ内単体で完全に動作する環境を構築できます。

--- 動作要件・前提条件 ---

1. Python 3.x がサーバーにインストールされていること。
2. 外部ライブラリとして `Jinja2` が必要です。
   ※インストールコマンド:
     pip install jinja2
3. サーバーが通常の CGI 実行 (.py 拡張子) をサポートしていること。
   ※常駐プロセス (WSGIやFlask等) を使用せず、リクエストごとに Python プロセスを起動する標準CGI方式で動作します。

--- ディレクトリ構造 ---

FFA_py/
  ├ index.html            (フレーム分割用のメインインデックスHTML)
  ├ frame1.html           (フレーム用の白紙HTML)
  ├ dev_server.py         (ローカル検証用の簡易CGIサーバープログラム)
  ├ config.py             (ゲームの各種環境設定・定数定義ファイル)
  ├ common.py             (データ保存・ロード、ロック処理、Jinja2制御共通ライブラリ)
  ├ templates/            (HTML表示用のJinja2テンプレートファイル群)
  ├ save_data/            (プレイヤー情報が保存されるセーブデータフォルダ)
  │  └ [ユーザーID]/       (ユーザー名ごとのフォルダ)
  │      ├ chara.json     (ステータスデータ：JSON形式)
  │      ├ item.json      (所持アイテム・装備データ：JSON形式)
  │      ├ syoku.json     (職業熟練度データ：JSON形式)
  │      └ souko_*.json   (倉庫に預けた武器・防具・装飾品データ)
  ├ html/                 (CSS、JS、静的ヘルプHTML)
  │  ├ cgi.css            (スタイルシート)
  │  ├ const.js           (右クリック禁止用JS)
  │  ├ ffhelp.html        (ゲームヘルプ)
  │  └ syokugyou.html     (転職職業ヘルプ)
  └ images/               (アバター画像などのゲーム素材フォルダ)
      ├ chara/            (キャラクターアバター画像 .gif)
      ├ choco/            (チョコボ画像)
      └ *.jpg, *.gif      (背景、その他素材)

--- 設置の手順 ---

1. 外部ライブラリの導入
   サーバー環境 (またはローカル環境) に `jinja2` をインストールします。
   > pip install jinja2

2. 環境設定 (config.py の編集)
   `FFA_py/config.py` 内をテキストエディタで開き、お好みのゲーム設定に変更します。
   - `b_time` (戦闘間隔秒数)
   - `mente` (メンテナンスモード 1: オン, 0: オフ)
   - `main_title` (ゲームのタイトル)
   - その他、経験値や上限設定、デザインカラーなど

3. サーバーへの転送とパーミッション設定 (Linuxサーバーの場合)
   FTP等でサーバーの公開ディレクトリに `FFA_py` フォルダごと転送します。
   - 動作用スクリプト (*.py)：「755」(またはサーバー推奨 of CGI実行権限)
   - 記録用フォルダ (save_data, lock)：「777」(またはサーバーの書き込み権限。所有者実行時は 707 や 755 でも可)
   - 静的ファイル (html/, images/ 内のファイル、index.html)：「644」

--- ローカル環境でのテスト・検証方法 ---

本パッケージには、開発・検証を容易にするための簡易CGIサーバー `dev_server.py` が同梱されています。
1. コマンドプロンプトやターミナルを開き、`FFA` または `FFA_py` ディレクトリに移動します。
2. 以下のコマンドを実行してテストサーバーを起動します。
   > python FFA_py/dev_server.py
3. サーバーが起動したら、ブラウザから以下のURLにアクセスします。
   > http://localhost:8000/FFA_py/index.html
4. これにより、ローカル環境（ポート8000）上で本番サーバーと同様のゲーム動作をテストできます。

--- セーブデータの移行とバックアップ ---

Perl版ではカンマ区切りのテキストファイルが複数のフォルダに分散していましたが、Python版では `save_data` フォルダに集約され、ユーザーごとにフォルダが作成されます。
- バックアップを行う場合は、`FFA_py/save_data` ディレクトリ全体を丸ごと退避させてください。
- プレイヤーデータは可読性の高い JSON 辞書形式になっているため、管理者による手動でのデータ調整や確認も容易です。
  例: `chara.json` を開いて `gold` の値をエディタで直接変更し保存する等。

--- よくある質問 (FAQ) ---

Ｑ. キャラクター作成やログイン時に `can't lock` や `ロックタイムアウト` と表示されます。
Ａ. 排他制御を行うための `lock` ディレクトリへの書き込み権限がありません。
    `FFA_py/lock` ディレクトリが存在するか確認し、そのパーミッションを `777` (またはサーバーが書き込み可能な設定) に変更してください。

Ｑ. 画面のデザインが崩れたり、キャラクター画像が表示されません。
Ａ. `html` フォルダや `images` フォルダが正しく配置されているか、パスが正しいか確認してください。
    各テンプレートファイルのアセット参照は、`FFA_py/html/` および `FFA_py/images/` を直接参照するよう最適化されています。
    CSSファイルや画像のパーミッション（通常 `644`）も確認してください。

Ｑ. Jinja2 レンダリングエラー (Jinja2 Rendering Error) が発生します。
Ａ. テンプレートファイル (*.html) が `FFA_py/templates` ディレクトリ内に正しくコピーされているか確認してください。また、サーバーに `jinja2` パッケージが正しくインストールされているか確認してください。
